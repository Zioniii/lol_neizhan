# 部署方案：混合架构

## 问题

当前系统战绩同步依赖本地 League Client（LCU）提供 JWT token 来调用 Riot SGP API，因此后端无法直接部署到服务器。

## 方案：服务器 + 本地同步脚本

```
服务器（公网IP）
├── FastAPI 后端
├── SQLite 数据库（共享）
└── 前端静态页面（所有玩家可访问）

每个玩家本地运行同步脚本
└── 连接本机 LCU → 拉取 SGP 数据 → 推送到服务器
```

### 服务器职责

- 选手管理、随机分组 → 无需改造
- 对局记录查询、胜率统计展示 → 无需改造
- 接收各玩家同步脚本推送的数据 → 需新增接口
- 托管前端静态文件 → 构建后部署

### 本地同步脚本职责

- 扫描本地 LCU lockfile
- 获取 JWT token
- 调用 Riot SGP API 拉取对局数据
- 推送到服务器 `/api/sync/push` 接口
- 无需启动整个后端，轻量运行

### 需要改动的代码

| 文件 | 改动 |
|------|------|
| `backend/routers/sync.py` | 新增 `POST /api/sync/push` 接口，无需 LCU 校验，只做数据入库 |
| `frontend/src/pages/Sync.tsx` | 改为向服务器发送同步请求（已无需后端 LCU） |
| 新增 `scripts/sync-agent.py` | 独立脚本：连接 LCU → 拉取 SGP → 推送到服务器 |

### 同步流程图

```
┌─────────────────────┐     ┌──────────────────┐     ┌───────────────┐
│  本地 League Client  │────▶│ sync-agent.py    │────▶│  服务器 API   │
│  (提供 LCU token)    │     │ (拉SGP+推数据)   │     │ /api/sync/push│
└─────────────────────┘     └──────────────────┘     └───────┬───────┘
                                                              │
                                                              ▼
                                                       ┌───────────────┐
                                                       │   SQLite DB   │
                                                       └───────────────┘
```

### 部署步骤（后续实施）

1. 服务器安装 Python + Node.js
2. 上传代码，安装依赖
3. 前端 `npm run build` 构建静态文件
4. 后端 `python -m backend.main` 启动服务
5. nginx 反代或直接暴露端口
6. 各玩家运行同步脚本
